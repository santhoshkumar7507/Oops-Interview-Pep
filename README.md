## 🎯 Core Concepts Covered

### The Four Pillars of OOPs
- **Encapsulation:** Hiding internal state and requiring all interaction to be performed through an object's methods.
- **Abstraction:** Hiding complex implementation details and showing only the essential features of the object.
- **Inheritance:** Mechanism where one class acquires the properties and behaviors of a parent class.
- **Polymorphism:** The ability of different objects to respond, each in its own way, to identical messages (Compile-time & Run-time).

### Advanced Topics
- **SOLID Principles:** The holy grail of object-oriented design.
- **Design Patterns:** Singleton, Factory, Observer, Decorator, Strategy, and more.
- **Class Relationships:** Association, Aggregation, and Composition.
- **Interfaces vs Abstract Classes:** When to use what and why.

## 💡 How to Use This Guide

1. **Start with the Core:** If you are a beginner, start by understanding the four pillars of OOPs.
2. **Understand the Design:** Move on to SOLID principles and common design patterns. These are highly tested in Senior/SDE-2 level interviews.
3. **Practice:** Go through the interview questions and try to solve the design problems before looking at the solutions.
